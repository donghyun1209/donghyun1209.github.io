  프로젝트 루트의 ToDayWork.md 파일을 읽어 Jekyll 블로그 포스트를 자동으로 생성하는 스킬.
  사용자가 "포스트 만들어줘", "오늘 작업 포스트로 올려줘", "ToDayWork 기반으로 포스트 작성해줘",
  "블로그 글 써줘" 등을 요청할 때 반드시 이 스킬을 사용한다.
  ToDayWork.md 내용을 분석해 제목·카테고리·태그를 추론하고, _posts/ 디렉토리에 Jekyll 포스트 파일을 작성한다.
---

## 목표

`ToDayWork.md`를 읽어 `_posts/YYYY-MM-DD-<slug>.md` 형식의 Jekyll 포스트 파일을 생성한다.

---

## 단계

### 1. ToDayWork.md 읽기

프로젝트 루트의 `ToDayWork.md`를 읽는다.
파일이 없으면 사용자에게 알리고 중단한다.

### 2. 메타데이터 추론

`ToDayWork.md` 내용을 분석해 다음을 결정한다.

**title**
- 파일 최상단 `#` 제목을 그대로 사용한다.
- 없으면 내용에서 핵심 주제를 뽑아 "프로젝트명: 핵심 작업" 형태로 만든다.

**date**
- 파일 내 날짜 표기(`> YYYY-MM-DD` 또는 날짜 언급)를 우선 사용한다.
- 없으면 오늘 날짜를 사용한다.
- 시간은 항상 `09:00:00 +0900`으로 고정한다.

**categories**
- 2개 이내로 결정한다.
- 프로젝트 이름(예: RasEyes, HelloWorld)과 기술 분야(예: Embedded System, Web, Algorithm)를 조합한다.

**tags**
- 본문에 등장하는 기술·도구·라이브러리·키워드를 소문자로 추출한다 (5~10개).

**slug**
- title에서 영문 소문자 + 숫자 + 하이픈만 남긴다.
- 예: "RasEyes Phase 4: Orange Pi 5" → `raseyes-phase4-orangepi5`

### 3. 포스트 본문 작성

기존 포스트 형식을 따른다.

**규칙**
- 한국어 문장과 영어 번역을 `\\`으로 구분해 같은 줄에 작성한다.
  ```
  한국어 설명.\\
  English explanation.
  ```
- 섹션 구분은 `---`를 사용한다.
- 코드 블록은 원본 그대로 유지한다.
- `ToDayWork.md`의 "다음에 할 것" 섹션은 포스트에 포함하지 않는다.
- 포스트 끝에 "마치며" 섹션 없이 마지막 기술 내용으로 끝낸다.

**front matter 템플릿**
```yaml
---
title: "제목"
date: YYYY-MM-DD 09:00:00 +0900
categories: [카테고리1, 카테고리2]
tags: [태그1, 태그2, ...]
---
```

### 4. 파일 저장

`_posts/YYYY-MM-DD-<slug>.md` 경로로 저장한다.

저장 후 다음을 출력한다:
```
포스트 생성 완료: _posts/YYYY-MM-DD-<slug>.md
제목: ...
날짜: ...
카테고리: ...
태그: ...
```

---

## 주의사항

- 날짜 시간은 항상 `09:00:00 +0900`으로 설정한다 (미래 시간으로 설정하면 GitHub Pages 빌드 시 포스트가 숨겨진다).
- 같은 날짜의 포스트 파일이 이미 존재하면 덮어쓰기 전에 사용자에게 확인한다.
- `ToDayWork.md`의 내용을 그대로 복사하지 말고, 블로그 독자를 위한 흐름 있는 글로 재구성한다.
